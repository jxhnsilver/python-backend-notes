**REST (Representational State Transfer)** — это архитектурный стиль проектирования распределённых систем, основанный на протоколе HTTP.
## 2️⃣ Основные идеи REST
 
 1. **Клиент–сервер**
	- Клиент отвечает за UI и отправку запросов.
	- Сервер отвечает за данные и бизнес-логику.
		
 2. **Ресурсы**
    - Всё — ресурс (пользователь, заказ, документ).
    - Ресурсы идентифицируются **уникальными URL**.
        
3. **Использование HTTP-методов**
    - `GET` → получить данные
    - `POST` → создать ресурс
    - `PUT` → заменить ресурс
    - `PATCH` → частично обновить ресурс
    - `DELETE` → удалить ресурс
        
4. **Stateless (без состояния)**
	- **Сервер не хранит информацию о клиенте между запросами.**
	- **`Каждый HTTP-запрос должен быть самодостаточным`** — содержать всю нужную информацию (токен, параметры и т.д.), чтобы сервер смог его обработать.
	- Сервер не "помнит", что было в предыдущем запросе клиента.
        
5. **Единообразие интерфейса (Uniform Interface)**
    - URL + методы + коды ответов = общий стандарт для взаимодействия.

## Что передаёт REST API

1. **URL (Uniform Resource Locator)** — путь до ресурса
	```
	https://api.example.com/users/123
	```
	Здесь `users` — ресурс, `123` — конкретный пользователь.
	
 2. HTTP-методы и примеры:
	- **GET /api/users** → список пользователей
	- **GET /api/users/5** → получить пользователя с ID=5
	- **POST /api/users** → создать пользователя
	- **PUT /api/users/5** → заменить пользователя с ID=5
	- **PATCH /api/users/5** → обновить часть данных пользователя
	- **DELETE /api/users/5** → удалить пользователя

## HTTP-коды ответов

- **2xx — успех**
    - `200 OK` — запрос успешен
    - `201 Created` — создан новый ресурс
    - `204 No Content` — успешно, но без ответа
        
- **4xx — ошибка клиента**
    - `400 Bad Request` — неверный запрос
    - `401 Unauthorized` — требуется авторизация
    - `403 Forbidden` — доступ запрещён
    - `404 Not Found` — ресурс не найден
        
- **5xx — ошибка сервера**
    - `500 Internal Server Error` — внутренняя ошибка сервера
    - `503 Service Unavailable` — сервис недоступен

## Что такое RESTful API

**RESTful API** — это API, которое **следует принципам REST**:

- Использует HTTP-методы по назначению
- Работает с ресурсами через URL
- Является Stateless
- Возвращает структурированные данные (обычно JSON)

## Суть REST

- Всё — **ресурсы** (users, orders, products).
- Каждый ресурс имеет **уникальный URL**.
- Взаимодействие идёт через **HTTP-методы**.
- Данные обычно в **JSON**.
- Сервер **не хранит состояние клиента**.
