**Models (Модели)** — это **Python-классы, которые описывают структуру таблиц в базе данных**.
## **КЛЮЧЕВЫЕ КОМПОНЕНТЫ МОДЕЛЕЙ**

### **1. Класс модели**

Каждая модель наследуется от базового класса `django.db.models.Model`. Это дает доступ ко всей функциональности ORM (Object-Relational Mapping) Django.

### **2. Поля модели (Fields)**

Поля определяют структуру таблицы. Каждое поле соответствует столбцу в таблице базы данных и имеет определенный тип данных.

### **3. Связи между моделями**

Модели могут быть связаны между собой через специальные типы полей, которые создают отношения в базе данных.

## **ТИПЫ ПОЛЕЙ (FIELDS)**

### **Основные типы полей:**

#### **Текстовые поля:**

- **CharField** — для короткого текста с ограничением длины
- **TextField** — для длинного текста без ограничения
- **EmailField** — для email-адресов с автоматической валидацией
- **URLField** — для веб-адресов

#### **Числовые поля:**

- **IntegerField** — для целых чисел
- **FloatField** — для чисел с плавающей точкой
- **DecimalField** — для точных десятичных чисел (важно для денежных значений)

#### **Дата и время:**

- **DateTimeField** — для даты и времени
- **DateField** — только для даты
- **TimeField** — только для времени

#### **Логические поля:**

- **BooleanField** — для значений True/False

#### **Файловые поля:**

- **FileField** — для загрузки файлов
- **ImageField** — для загрузки изображений (требует Pillow)

## **ПАРАМЕТР on_delete**

Параметр `on_delete` определяет, что происходит с связанными записями при удалении основной записи:

### **Варианты поведения:**

1. **CASCADE** — удалить все связанные записи
2. **PROTECT** — запретить удаление, если есть связанные записи
3. **SET_NULL** — установить значение NULL (требует `null=True`)
4. **SET_DEFAULT** — установить значение по умолчанию
5. **SET()** — установить значение через функцию
6. **DO_NOTHING** — ничего не делать (может вызвать ошибки базы данных)

## **ДОПОЛНИТЕЛЬНЫЕ ОПЦИИ ПОЛЕЙ**

### **Общие параметры:**

- **null** — может ли поле быть NULL в базе данных
- **blank** — может ли поле быть пустым в формах
- **default** — значение по умолчанию
- **unique** — должно ли значение быть уникальным
- **verbose_name** — человеко-читаемое имя поля
- **help_text** — текст подсказки для форм
- **choices** — ограниченный список допустимых значений
