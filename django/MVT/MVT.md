**MVT (Model-View-Template)** — это **архитектурный паттерн Django**, который состоит из трёх основных компонентов:
## 1. **Model** - Модель

- **Роль**: Работа с данными
- **Что делает**: Определяет структуру данных в БД
- **Пример**: Классы Doctor, Clinic, Schedule, Talon в вашем коде
## 2. **View** - Представление (Контроллер)

- **Роль**: Обработка запросов и бизнес-логика
- **Что делает**: Получает запрос → работает с Model → передает данные в Template
- **Пример**: Функции в ваших `*_views.py` файлах
## 3. **Template** - Шаблон

- **Роль**: Отображение данных
- **Что делает**: HTML с динамическими данными от View
- **Пример**: Ваши HTML файлы в папке `templates/`
## Как они работают вместе:

```text
Пользователь → URL → View → (Model → БД) → View → Template → Пользователь
```
