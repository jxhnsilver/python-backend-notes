## Что это?

- Мьютекс (блокировка) внутри CPython, который **разрешает выполнять Python-байткод только одному потоку за раз**.
- Защищает внутренние структуры интерпретатора (рефсчётчики, куча) от гонок.

## Что НЕ блокирует GIL?

- Операции ввода-вывода (чтение файла, сетевые запросы) — GIL **отпускается**.
- Вычисления в C-расширениях (NumPy, pandas, async I/O) — если они явно отпускают GIL.

## Последствия

- **Потоки в Python не дают прироста производительности для CPU-bound задач**.
- Для CPU-heavy нагрузки → используй `multiprocessing`.
- Для I/O-heavy (API, БД) → потоки и async работают отлично.

## Async vs Threads

- `async/await` — однопоточный, но без блокировок → идеален для I/O.
- Потоки — удобны, но «обманчивы» из-за GIL.

## Мифы

- ❌ «GIL можно выключить» → только в других реализациях (PyPy, Jython).
- ❌ «GIL делает Python медленным» → только для CPU-bound, не для типичного бэкенда.
